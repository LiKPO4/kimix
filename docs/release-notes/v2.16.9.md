# Kimix v2.16.9

本版本修复长时间工具执行过程中错误显示“输出完成”，以及运行中旧思考和工具调用突然消失的问题。

## 修复

- 本地 wire 历史只把 `step.end(end_turn)` 视为整轮结束；`tool_use` 仅作为同一轮内的模型/工具步骤边界，不再提前关闭 Assistant。
- 运行中的 Server snapshot 若缺少本地已经收到的工具和过程事件，不再因为最新思考文本不同而覆盖更完整的本地时间线。
- 最新回合只要官方 runtime 仍处于运行状态，渲染层就保持未完成状态；已成功的阶段性工具不会让过程标题闪成“输出完成”。

## 验证重点

- 连续执行多个读取、搜索和命令工具时，标题应持续显示运行态。
- 新工具到达后，较早的思考和工具调用仍应保留；超过详情渲染上限时应出现明确的“显示全部”入口，而不是静默消失。
