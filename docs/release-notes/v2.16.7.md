# Kimix v2.16.7

本版本修复 Electron renderer 重新加载后可能永久停在“正在准备默认项目”的问题。

## 修复

- 启动上下文由窗口首次加载的一次性通知改为每个 renderer 文档完成加载时重新发送；开发热重载、手动重新加载和空白页自恢复后都能重新取得项目与会话上下文。
- bootstrap 监听在页面加载前完成注册，避免快速本地文件加载越过恢复监听器。
- 默认项目目录只在应用生命周期内延后准备一次；最近项目记录改为 bootstrap 发送后的附属持久化，同一项目重载不再反复写盘，失败后仍可重试。
- 默认项目准备严格排在恢复 payload 发送之后；Windows 开发环境会先规范化比较路径，避免仅目录大小写不同却重复遍历同一默认项目。
- 默认项目迁移、目录准备或最近项目记录失败不再阻断界面恢复，可捕获的异常会写入日志。
- 空白页检查绑定 renderer 文档代次，并取消旧定时器，避免快速连续重载时由旧检查误判新页面。

## 验证

- 新增主进程启动握手回归，覆盖全新 preload replay channel 下的第二次文档加载、默认项目一次性延后准备、最近项目写入去重与失败重试，以及安全默认项目降级。
