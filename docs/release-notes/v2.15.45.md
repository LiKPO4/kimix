# Kimix v2.15.45

## 恢复多 Agent 房间入口

- 修复本机实验开关记录缺失时被误判为关闭，导致 `添加 Agent`、接收者选择和正文补充控件同时消失的问题。
- 多 Agent 房间入口现在默认开放；用户明确关闭时会持久化关闭值，不再把“没有记录”和“主动关闭”混为一谈。
- 已经存在的房间不再受创建入口开关影响，成员选择、上一轮正文补充和房间路由始终保持可用。
- 房间发送链路只依据会话是否具有 collaboration 数据，不会因为 UI 开关异常而退回普通单 Agent 发送。
- 当本地 collaboration 分组已经丢失，但官方 session metadata 能唯一对应到现存主会话时，Kimix 会自动重建房间骨架并重新加载各 Agent 的官方历史；旧 metadata 未保存昵称的成员会先以 `Agent 2/3/4` 显示，模型信息恢复后仍可重命名。
- 持久化层会记住已经确认的房间结构；后续普通 snapshot 只能更新主 Agent 镜像，不能再把整个房间覆盖成单 Agent 会话。未来 schema 或归属冲突仍保留在“待找回 Agent”，不会冒险自动合并。
