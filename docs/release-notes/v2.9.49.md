# Kimix v2.9.49

## Kimi Code 0.14.2 跟进

- 跟进 `tool.progress` 事件，Bash 等 shell 工具运行中会把 stdout/stderr 合并到同一个工具卡，展开即可查看实时输出。
- 补齐 `/reload`、`/status`、`/usage` 本地 slash 到 SDK API：分别重载当前 SDK 会话、显示当前会话状态、显示当前会话用量。
- 确认 `/swarm` 当前已走官方 SDK：`/swarm on`、`/swarm off` 和 `/swarm <任务>` 调用官方 `Session.setSwarmMode()` / `Session.swarm()`，不再作为普通消息发送。
- 接入官方 `getConfigDiagnostics()`，在插件 / Skills 页显示配置警告；SDK Skill 列表展示 Sub-skill 标识和数量统计。
- 历史回放兼容 SDK 原生 `assistant.delta`、`thinking.delta`、`tool.progress`、`turn.ended` 和 `compaction.*` 事件，避免上下文压缩 replay 后丢失后续输出或工具进度。

## 运行态修复

- 修复上下文压缩完成后，当前轮次仍在执行但输入框提前解锁的问题；现在等待官方 SDK `turn.ended` / status completed 收口。
- 修复官方 active turn 残留/恢复场景下新消息误发后 UI 再次清运行态的问题；官方拒收的新 turn 会回滚本地占位，队列消息放回待发队列。
- `sendKimiCodePromptWithRetry` 不再在多次 active-turn 拒绝后自动取消官方当前轮次，避免误停真实仍在执行的任务。
- 修复 Kimi Code 实时输出文字片段重复的问题：旧兼容事件通道 `kimi:event` 不再处理 Kimi Code/长程任务会话，避免与 `kimi-code:event` 新通道同时合并同一批 SDK 原生 delta。

## UI 与交互修复

- 修复侧栏底部「设置」入口 hover 背景色与上方「插件 / Hooks」等导航按钮不一致的问题，统一使用 `surface-hover` 主题 token。
- 统一高优先级按钮 hover 反馈：停止按钮、弹窗强按钮、审批允许按钮不再用透明度变淡作为 hover。
- 修复画板删除对象、附件删除、MCP OAuth 开关、顶部工具栏组合按钮的 hover 色体系不一致问题。
- 优化长会话滚动到底部的顿挫感：消息区底部保留空间从 120px 收敛到 60px，同时保留底部呼吸感。
- 收窄 `scripts/restart-kimix-dev.ps1` 的进程匹配范围，只处理命令行包含当前仓库路径的 dev 进程，避免误碰安装版或其他 Kimix 相关进程。
- 版本号三处同步到 v2.9.49：`package.json`、侧栏、设置页。
