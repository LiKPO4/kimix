# Kimix v2.15.56

## 以稳定投递身份消除房间消息重复投影

- 每次房间投递现在通过现有 Kimix 封套携带 `roomMessageId`、`agentTurnId` 和 `dispatchAttemptId`，运行事件与重载后的官方历史使用同一组身份。
- 房间时间线只渲染一条共享用户消息；每个 Agent 的官方 `TurnBegin` 作为该投递的精确回合锚点，不再因 snapshot、Agent 切换或重启变成第二条气泡。
- 两次完全相同的输入也会按各自投递身份分别归属，不依赖正文或时间窗口猜测。
- 没有稳定身份的旧历史继续使用“同一 Agent、正文一致、30 秒内且唯一候选”的保守迁移；多个候选不会自动合并。
- 备份冲突副本会连同投递尝试身份一起重映射，避免复制后的事件引用原房间。
