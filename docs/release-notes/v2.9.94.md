# v2.9.94

## 性能修复：会话切换与启动卡顿

### 问题

切换旧会话时 UI 卡顿 3-4 秒；启动时加载活跃会话也有明显卡顿。

### 根本原因

1. `MarkdownRenderer` 未被 `React.memo` 包裹，每次父组件更新都会重新解析 Markdown + 语法高亮，代价极高。
2. 切换会话时 `updateSession` 将完整 events 数组写入全局 store，触发 ChatThread 一次性渲染最多 28 个 `MarkdownRenderer`，同步阻塞 JS 线程 3-4 秒。

### 修复

- **`MarkdownRenderer` 添加 `React.memo`**：用 `content` 和 `wrapLongLines` 做精确比较，避免内容未变时重复渲染。会话内流式输出、工具调用等非 Markdown 变更将不再重渲染历史消息。
- **ChatThread 分阶段渲染**（Phase Render）：
  - 切换会话时先渲染最近 5 条消息（`CHAT_INITIAL_RENDER_ITEMS = 5`），立即显示内容（<200ms）。
  - 通过 `React.startTransition` 在后台展开到完整 28 条（`CHAT_FULL_RENDER_ITEM_LIMIT = 28`），不阻塞主线程，用户可继续操作。
  - Phase 2 渲染完成后自动滚动到底部，保证视图一致性。

### 效果

- 会话切换第一帧显示时间：~200ms（原 3-4 秒）
- 全量内容加载后台完成，期间 app 保持可交互
- 同一会话内流式消息更新不再触发历史消息重渲染
