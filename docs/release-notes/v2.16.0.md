# Kimix v2.16.0

## 多 Agent 房间

- 普通 Kimi Code 会话可直接添加独立 Agent；每个 Agent 使用原有 Provider、模型、权限和独立官方上下文，单个房间最多四个 Agent。
- 支持单选、多选与 `@Agent` 临时覆盖接收者；多个空闲 Agent 可并行处理，排队、停止、引导、审批、提问和用量均按 Agent 独立归属。
- 默认向目标 Agent 补充上一轮可见正文，也可单次选择最近三轮、指定消息、全部正文或不补充；只共享用户正文和各 Agent 最终正文，不共享隐藏思考、工具过程或审批数据。
- 搜索、Markdown 导出、官方调试包导出、归档、恢复、通知和上下文用量均支持房间内的逐 Agent 边界。

## 可靠性与体验

- 为房间消息、Agent 回合和每次投递建立稳定身份，修复运行中与重启后用户消息重复、历史归属漂移、旧状态覆盖最终用量及耗时跨回合的问题。
- 投递在调用官方运行时前耐久保存；并发写入会等待最新合并快照真正落盘，未知确认状态不会自动重发。
- 外部终止或重启后按持久化投递恢复并校准各 Agent 运行态；失效 runtime 可安全停止，不影响其他 Agent。
- 输入区工具栏完成紧凑化与统一基线；长思考超过六行后使用低对比度内部滚动区；Windows 完成通知可恢复并聚焦对应 Kimix 窗口。

## 使用提示

- 房间内所有 Agent 共享同一项目目录。Kimix 不自动创建 worktree、加文件锁或回滚冲突；并行修改文件时请通过接收者选择和提示词明确分工。
