# Kimix v2.16.6

本版本根治多 Agent 房间中单次发送偶尔显示为两条用户消息的问题。

## 修复

- 修正旧主会话兼容回写会重建 primary delivery、从而丢失 `dispatchAttemptId` 和其他投递事务信息的问题；同一事务现在增量合并并保留完整审计字段与原接收者顺序。
- 已受损的历史房间会在加载时按同一 Agent 的精确房间消息、回合及唯一 attempt 自动恢复；存在不同 attempt、跨投递占用或不完整显式身份时不会猜测。
- 时间线与 canonical history 共用事务身份解析：事务身份优先，完全无身份的旧 official event ID 仅在该 Agent 内唯一时兼容绑定，正文与时间只保留给双向唯一的旧历史迁移。
- 事件认领改为 Agent 作用域，避免不同 Agent 恰好复用相同官方事件 ID 时互相吞掉历史。
- 相同正文但属于不同稳定投递的真实消息继续分别显示，不进行正文去重。
