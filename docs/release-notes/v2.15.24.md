# Kimix v2.15.24

## 多 Agent 精确路由与独立队列

- 房间 `@` 补全改为真实成员；`@Agent` 按正文出现顺序精确路由，大小写不敏感，未识别 mention、插件 mention 和 email 文本不会被误删。
- 接收者菜单支持多选作为默认目标；正文中的一个或多个 `@Agent` 会临时覆盖默认选择，路由 token 只从发送给模型的冻结 payload 中移除，用户可见消息保持原文。
- 一条房间消息为每个接收者创建稳定 delivery 和响应块；空闲 Agent 并发启动，忙碌 Agent 只排自己的队列，不阻塞其他参与者。
- 每个 Agent 每次只发送最早 queued delivery；当前轮终态后自动续发下一条，应用重启仍沿用既有 sending 核验与 indeterminate 防重复策略。
- queued 响应块显示具体 Agent 和“取消排队”；failed 或 indeterminate delivery 只有用户显式点击重试才创建新的 attempt/turn。
- 多 Agent 响应块按冻结接收者顺序展示，并在过程标题显示 Agent 名称，异步完成顺序不会重排 UI。
