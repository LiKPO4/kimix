# Kimix v2.17.0

本版本是 **v2.16.88 之后的中等版本更新**：底座跟进 **Kimi Code 0.29.0**，补齐会话级子 Agent 模型、主 Agent 思考强度、视频多模态、历史用量页脚与一批渲染/设置稳定性修复。正式包仍由 **GitHub Actions** 构建发布。

## 运行时与协议（0.29）

- 本机官方 CLI / vendored Node SDK 底座对齐 **0.29.0**（node-sdk 0.14.0），并保留子 Agent 双模型实验补丁（PR #1996 六提交 + Kimix sticky resume / spawn 审计）。
- 接入官方 **工具 `active` 状态**、**Agent 生命周期诊断**（`agent.created/disposed` + 快照 subagents）。
- 主 Agent **思考强度**可按模型 `support_efforts` / `default_effort` 选择；自定义模型可在设置中**声明**思考档位（官方探测不回档位时以 config 为准）。
- 会话侧栏支持 **子 Agent 默认模型 / 思考强度**；明确独立模型时走兼容路由，并在 fresh runtime 上**重放已存路由**（重启后不再静默回落跟随主模型）。
- 强制委派开关：开启后仅在 **wire 发送层**幂等注入委派指令，可见气泡正文不变。
- Esc / 中断：`prompt.aborted` 纳入等待器，dispatch 立即结算（双路由回归已验证）。

## 多模态与媒体

- 支持粘贴 / 选择 / 拖入 **视频** 作为官方多模态内容（Server 先上传 file，再以 video+file 发送；兼容 SDK 仍可接收 `video_url`）。
- 历史官方视频经 **`kimix-media` + 0.29 `fs:content`（ETag/Range）** 流式播放，失败可回退整段加载；fileId 限制在 `~/.kimi-code/files` 下。

## 对话渲染与用量

- 思考流：合并幂等、快照回放**替换而非追加**、子代理事件不再并入主 turn 草稿。
- kimi-web 过程展示：思考保持官方内联 ThinkingBlock 风格（与工具组卡区分）。
- 历史轮页脚：从 wire 镜像水合模型 / token；canonical 被拒时增量合并用量状态；头部按**轮级真实模型**显示。
- 兼容链路会话历史**不再读 Server 休眠快照**，避免「模型请求失败」误报帧。
- 回合 settle 后补挂模型/用量；Composer 上下文环与弹窗与页脚共用合并逻辑（无官方 context 字段时用本轮输入近似占用；压缩/新用户轮后不粘住旧窗口）。

## 模型与设置 UI

- 供应商 **Base URL 探测模型** 与添加流程相邻；去掉易混淆的官方目录模型下拉选模。
- 过滤 `models.<alias>.overrides` 伪模型；删除模型时清理残留。
- 健康卡「需关注」可见 Server 徽章与具体项；权限/思考弹窗更紧凑；探测卡与加号/语音按钮圆角对齐。
- 窄宽度工具栏：优先收权限为图标保持一行，再换两行。
- 思考 / 权限 / Plan 在会话不可用时软失败（保留本地值，resume 统一重放）。

## 其他

- 文件变更统计与历史预览补全；重试轮历史隔离；流式正文片段顺序保持。
- 安装列表在 GitHub API 限流时从 `latest.yml` 恢复。
- 知识库补充 runtime 不变量（子 Agent 路由、思考合并、用量水合、0.29 工具/生命周期等）。

## 已知边界（请知晓）

- 子 Agent **独立模型**依赖实验双模型路由；与官方 **自定义 Markdown Agent（agents/*.md）** 在 0.29 上不可兼得（内置 explore/coder/plan 不受影响）。
- 部分轮次官方不提供 `contextTokens` 时，Context 百分比为**基于本轮输入的近似**。
- 被污染的历史会话事件不会自动回刷；请用新会话验证误报/思考重复类问题。
- vendored SDK 仍携带未合并上游 PR #1996；刷新 vendor 需重挑提交。

## 验证

- 定向与相关单测通过（sessionMetrics / ChatThread / 渲染与路由等）。
- TypeScript 严格类型检查通过。
- 知识库 `pnpm knowledge:validate` 以仓库脚本为准。
- Windows / macOS / Linux 正式产物由 GitHub Actions 构建。

## 建议重点复验

1. 新会话：主模型思考档位切换、自定义模型声明档位是否出现在菜单。
2. 子 Agent 卡：指定独立模型后发一轮 Swarm/explore，重启应用再发，子代理不应回落主模型。
3. 粘贴小视频发送与历史会话点击播放（拖动进度）。
4. 含子代理的长轮结束后页脚应出现模型 + 输入/输出（及 Context）；输入区上下文弹窗不再长期「等待上下文数据」。
5. 兼容路由会话：不应再出现无故「模型请求失败」失败帧。
6. kimi-web 模式下思考为内联次要色文案，而非「思考过程」工具式组卡。
