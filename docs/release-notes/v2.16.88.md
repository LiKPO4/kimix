# Kimix v2.16.88

本版本聚焦 **Agent 流式输出时的全局卡顿与正文正确性**，并跟进 Kimi Code 0.28、消息头/失败轮次/撤回等稳定性修复。流式性能经 CDP CPU 采样定位到真实碎任务热点后完成收口。

## 性能与流畅度

### 流式输出卡顿（主修复）

- **侧栏/会话过滤**：`isHiddenInternalSession` 不再对每个会话全量扫 events 跑正则；按 session 引用缓存，只检查标题与前几条用户消息，消除「无长任务却主线程假死数秒」的碎任务风暴。
- **贴底 settle 空转**：`settleSessionAtBottom` 在已贴底且布局稳定时提前结束，避免每 80ms 空写 `scrollTop` 并刷 `writeDiag` IPC。
- **流式写入路径**：活跃轮 text/thinking 写入局部 draft store（不每 token 撼动整棵会话树）；draft 通知合帧约 10fps；流式 plain Markdown（不做 katex/高亮全文 lex）；滚动时 flush 让路。
- **投影与渲染**：协作时间线身份保持投影 + 输入引用短路，历史已完成轮不再每 flush 全量重算；助手过程区与正文拆分子树 memo。
- **批处理与落盘**：工具参数流、status 等高频信息事件回到 80ms 批；流式活跃期本地持久化降频（结束/切后台再落盘）；running-sample 全量历史对账触发阈值放宽。
- **实时思考**：过程展开时只渲染思考尾部，降低文本 shaping 成本。

### 流式正文正确性

- 完成/barrier/稳定快照等**权威全文帧**到达时丢弃 draft，并对 open assistant **替换正文**，避免「你好霖江路」等问候与前半段重复拼接。
- draft 对累积帧做前缀安全合并；纯 token delta 仍正常拼接。
- 活跃正文 draft 的 turn 键可回退到 room activity 的 `activeTurnId`，减少「正在输出却几乎看不到正文」。

## 稳定性与协议

### Kimi Code 0.28

- 跟进 0.28 启动参数（含 `kimi web --no-open`）与权限文案。
- Vendor SDK 升至 0.28；多实例 Server 端口策略（读实例配置、占用时递增换口）。

### 消息头、失败轮次与过程区

- 终端 settle / 轮询：无正文前不误删空 placeholder；启发式 settle 不再强制收口仍活跃的轮次。
- 失败轮次：live 失败帧自构与可见性、启动后恢复失败轮次。
- 多步工具轮：completion barrier 跨步保留思考段，不再只剩「N 个工具调用」。
- 输出中手动展开过程区后，不再因换泡重挂载被重置折叠。
- 迟到的官方回放消息按官方序号插入，不再盲目追加到末尾。

### 撤回与模型

- 已派发但无应答的消息撤回时同步官方 undo，避免重发后模型看到重复内容块。
- 收敛会话模型所有权并锁定 prompt 模型；prompt 完成增加权威消息交付屏障。
- 重做第三方模型供应商管理；模型菜单切换时弹窗不横移，按钮恢复自适应宽度。

- 知识库新增 streaming-render-pipeline 架构说明。

## 建议重点复验

- 长会话边输出边滚动、点菜单：应明显比 2.16.56 更跟手，无数秒整页冻住。
- 流式过程中正文应持续增长；完成后问候/前半段不重复，Markdown 结构正常。
- 多工具多步轮展开过程：思考与工具同时可见。
- 撤回「已发送但无回复」的消息再重发：模型上下文无双份相同块。
- Kimi Code 0.28：冷启动 Server、多开实例换端口。
