# Kimix v2.16.56

本版本继续收口长对话历史与视口稳定性问题，重点根治“多个 Agent 回复被合并到同一轮”的持久化污染，并优化长会话首次打开时的可见范围。

## 修复

### 多轮回复错误合并

- 官方快照消息的稳定 ID 现在是不可跨越的消息边界；80ms 事件批处理、历史映射和时间线合并均不会再把不同官方消息压成一条。
- 即使历史回放窗口缺少原始用户边界，未见过的稳定消息也会独立挂载，不再落入“最近未完成回复”的通用合并路径。
- cache v9 会自动修复旧版本已经写入本地的跨轮污染。正式启动暂时使用无稳定 ID 的 SDK/wire 回退历史时，也只在本地稳定身份、多行结构和 80% 以上跨轮完整正文证据同时成立时采用较短官方历史，避免误删正常引用。
- 修复 canonical 候选尚未真正采用就提前标记迁移完成的问题；被拒绝的旧缓存会保留待修复版本，后续打开仍会继续尝试自愈。

### 会话打开与滚动

- 修复首个自动打开会话在真实滚动节点挂载前提前完成置底初始化，导致轻滚后突然上跳的问题。
- 普通滚轮和触摸滚动不再隐式扩展隐藏历史，历史只通过明确的展开或导航请求挂载，避免一次手势同时改变滚动位置和 DOM 高度。
- 初次打开长会话时，以用户消息为轮次边界显示最近 5 个完整轮次；典型对话由原先 4 项（约 2 轮）提升为 10 项。
- 折叠历史入口贴近对话区顶部，顶部留白由约 46px 收敛为 14px，同时保留按钮所需呼吸空间。

## 数据自愈验证

- 实际受影响会话从 cache v7、152 个事件、36204 字符迁移为 cache v9、150 个事件、22422 字符。
- 13782 字符的跨轮污染消息已移除；完整关闭并再次打开正式应用后，干净历史保持不变。

## 建议重点复验

- 打开曾出现多轮回复混合的长会话，确认每轮回复独立且重启后保持正常。
- 新发一轮消息，确认历史回复不会被重新合并到当前轮。
- 检查首次打开时左侧导航刻度覆盖最近 5 轮，滚到顶部后折叠历史入口紧贴对话区顶部。
