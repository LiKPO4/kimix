# Kimix v2.8.336 Release Notes

## 大版本适配：Kimi Code 0.11.0、官方 Goal 与消息流体验

本次更新汇总了 v2.8.295 之后的连续改动，重点完成 Kimi Code 0.10.x / 0.11.0 的 SDK 适配、官方 Goal 的 Kimix 图形化接入，以及多处对话流、侧栏和 Markdown 展示修复。

### Kimi Code 0.11.0 适配

- 刷新 vendored Kimi Code node-sdk 到官方 0.11.0 对应源码，继续使用 Kimix 内置 bundle，避免依赖外部 research checkout。
- 确认并适配 `createKimiHarness()` 优先、`KimiHarness` fallback 的 SDK 加载方式。
- 自动开启官方 Goal 所需实验开关，并跟进 0.11.0 的子 Skill 发现能力开关。
- 设置页增加 `kimi doctor` 诊断入口，方便排查 Kimi Code 环境、登录和模型配置问题。
- Windows 环境增加 Git Bash 预检提示，减少 Kimi Code 启动时的环境问题。

### 官方 Goal 接入

- 接入官方 `Session.createGoal / getGoal / pauseGoal / resumeGoal / cancelGoal` API。
- 右侧会话侧栏新增“官方 Goal”卡片，支持刷新、启动/替换、暂停/继续、取消。
- 输入框上方新增一行 Goal 状态条，展示当前目标、状态和轮次；支持关闭，并可从右侧侧栏恢复显示。
- `/goal <目标>` 与 `/goal replace <目标>` 会同步进入聊天消息流，并触发 Kimi Code 继续执行当前官方 Goal。
- `/goal status` / `/goal show` 会在对话流中追加本地状态反馈，不再只刷新侧栏。
- Goal 运行中会随 Kimi Code 消息流实时刷新轮次和状态；终态会优先采信 `UpdateGoal` 工具成功结果，避免完成后侧栏继续挂旧目标。
- 明确 `/goal next` 的边界：Kimi Code 0.11.0 TUI 已修复队列体验，但当前 node-sdk 仍未公开目标队列 API，Kimix 不伪造真实队列。

### Slash 命令与对话流

- 恢复 Kimi Code SDK 会话的本地 slash 建议，并拦截 `/goal`、`/compact`、`/plan`、`/btw`、`/undo`，避免这些本地命令作为普通文本误发给模型。
- 本地处理 slash 命令时，在聊天流中央追加浅蓝提示，让用户知道指令已被 Kimix 接收。
- `/compact` 成功路径保持静默，失败时才显示错误。
- 修复发送 Goal slash 后聊天流不显示用户消息、旧 session 覆盖新消息、运行中再次输入触发 active turn 错误等问题。
- 修复纯工具轮只有 `UpdateGoal` 执行体、缺少 assistant 轮次头部的问题。

### 会话列表与历史恢复

- 修复旧 Goal 会话打开后消息为空、左侧列表不显示当前可打开会话的问题。
- 左侧项目会话匹配改为规范化路径比较，兼容大小写、斜杠和尾斜杠差异。
- 左侧栏支持多个项目同时展开，方便多线程工作时查看各项目下会话的运行状态。
- 修复多项目展开后点击已展开项目无法收起的回归。
- 修复打开旧 Goal 会话时可能出现 React `Maximum update depth exceeded` 的循环更新错误。

### 模型与设置

- 修复设置页切换默认模型后被旧缓存覆盖的问题。
- 启动或恢复 Kimi Code 会话时统一使用当前默认模型，恢复旧会话后也会显式调用 SDK `setModel()`。
- SDK `setConfig()` 后增加 `getConfig({ reload: true })` 复读校验；失败时回退 TOML 写入。
- 底部模型显示优先跟随当前会话和当前默认模型，减少“底部 AI 与实际 AI 不一致”的情况。

### Markdown 与导出

- 修复 Goal 进度正文多段内容被压成一个段落的问题，聊天流和导出 Markdown 都会保留更合理的段落分隔。
- 修复流式/历史内容把 GFM 表格分隔行切碎后无法渲染为表格的问题。
- 新增相关单测覆盖段落恢复、表格 separator 恢复和 Goal 状态推断。

### 更新与界面细节

- 更新记录弹窗中的 Kimi Code “浏览器查看”改为打开官方 changelog，避免进入 403 页面。
- Kimi Code 更新卡片的“重新安装/更新”和“检查 Kimi Code”按钮高度与顶线统一。
- 顶部工具栏统一项目、终端、差异面板、会话侧栏等图标按钮的 hover/active 反馈。
- 顶部项目文件夹图标改为与同组工具按钮同色，hover 时统一加深，不再从黄色跳到黑色。
