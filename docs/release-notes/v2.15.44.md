# Kimix v2.15.44

## 移除旧需求澄清包装

- 移除 Composer `+` 菜单中历史遗留的“需求澄清”三态开关，以及对应的本地状态、设置同步和 IPC 配置字段。
- 普通消息、房间消息和斜杠命令不再被 Kimix 改写为“需求澄清说明 + 用户原始需求”，发送内容保持用户原文。
- 历史映射兼容剥离新旧两种需求澄清包装，官方 snapshot 回放后仍只显示用户真正输入的消息。
- 本地 IndexedDB 会话读取时也会迁移旧包装；即使暂时无法取得官方历史，旧用户气泡仍可恢复为原始需求。
- Kimi 历史缓存版本升级到 4，并把遗留澄清包装列为 canonical 修复证据；已缓存的错误气泡会主动接受官方原文纠正。
- 官方结构化提问卡、等待回复通知和长程任务主动提问能力继续保留，不受旧开关移除影响。
