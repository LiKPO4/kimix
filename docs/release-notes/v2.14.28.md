# Kimix v2.14.28

## 修复

- 优化 `/skill:...` 缺失时的 Skill 注册表刷新链路。
- 新安装或同步的 Skill 不在当前会话可见时，Kimix 会优先对同一个会话 ID 执行 reload 并切到 SDK 兼容路由后再激活 Skill，尽量避免创建 `skill-*` fork 会话。
- 如果同会话 reload 不可用、当前轮仍在运行，或 reload 后 Skill 仍不可见，仍保留原有 fork fallback，避免阻断用户调用 Skill。

## 验收建议

- 在已有会话里调用一个当前会话尚不可见的本地 `/skill:...`。
- 期望优先留在同一个会话 ID 内完成 Skill 激活，不新增同名 `skill-*` 会话。
- 若 reload 失败，应仍能 fallback 到旧的 fork 方案并给出可见错误或状态。
