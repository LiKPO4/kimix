# Kimix v2.9.20

## 主要更新

### 全局主题系统

- 新增全局色彩方案体系，内置暖纸、灰白、淡绿、暖橙、自定义等主题套装，并同时适配浅色、深色和跟随系统模式。
- 新增 KIMI 主题套装支持，可扫描 Kimi Code 官方主题 JSON，并以 `KIMI-主题名` 的形式登记为多套可切换主题。
- KIMI 主题支持官方 18 个颜色 token，不再只做简单三色映射。
- 设置页支持删除已登记的 KIMI 主题；删除只移除 Kimix 登记，不删除官方主题文件。
- `/custom-theme` 在 Kimix 内提供 SDK 兼容执行路径，会生成官方 Kimi Code 主题 JSON，生成后可在设置页点击“扫描官方主题”导入。
- 自定义主题编辑区统一为 KIMI token 风格的色块和十六进制输入样式。
- 强化 `/custom-theme` 兼容提示，要求先解析真实用户目录再写入 `.kimi-code/themes`，避免主题文件写到错误用户目录。

### Kimi Code 0.12 / 0.13 跟进

- 补充 `/custom-theme`、`/import-from-cc-codex`、`/swarm`、`/compact` 等官方 slash 指令展示和模板；其中 SDK 暂不支持 API 调用的 `/swarm` 会在提示中明确标注，发送后按普通消息处理。
- `/import-from-cc-codex` 支持安全预览和确认导入流程，可导入 Claude Code / Codex 的 instructions、skills 和 MCP 配置。
- 修复完整 slash 指令在回车时被补全菜单吞掉的问题。
- 跟进 Kimi Code 安装包下载地址迁移，更新 Windows 安装清单和二进制下载路径。
- 模型设置诊断增加 Kimi Code Home、代理变量和微压缩状态说明。
- OpenAI-compatible 外部模型支持从设置页删除；Kimix 会备份 `config.toml`，并在无引用时一并移除对应 Provider。
- 底部状态栏和模型状态胶囊展示模型名时，会自动隐藏斜杠前缀，只显示最后一段模型名称。
- 统一底部状态栏项目、分支、模型、连接状态、套餐用量和导出按钮的鼠标悬停阴影与背景反馈。
- 修复启动或 Ctrl+R 重载时先闪烁默认主题、再切换到已选主题的问题。
- 修复引导消息图片丢失、状态文案过早显示为已引导，以及引导消息被错误挂到助手回合顶部的问题。
- 修复底部状态栏模型名等英文下探字符被裁切的问题。
- 优化思考过程分段逻辑，避免 Kimix 在长度阈值处把同一句话从中间切开。
- 修复引导状态和消息边界：只有收到官方 `turn.steer` 事件后才显示“引导成功”，并让后续助手消息从引导气泡下方重新开始。
- 修复 Kimi Code 实际回放事件使用 `turn.steer` 而非 `SteerInput` 时，Kimix 未识别成功标志，导致引导消息一直停留在“引导中”的问题。
- 修复 live 引导场景中不能把 `session.steer()` 返回误当作引导成功的问题；Kimix 现在等待官方 `wire.jsonl` 中匹配的 `turn.steer` 落盘后，才把本地引导气泡改为“引导成功”。
- 修复引导成功后上方旧 assistant 段仍继续计时的问题；官方 `turn.steer` 确认时会封口上一段，并把后续计时交给下方新起的 assistant 段。
- 修复打开旧对话时，本地 slash 指令接收提示可能漂到 agent 回复下方的问题；现在提示会绑定并显示在对应用户消息下方。
- 隐藏套餐用量浮层中已失效的“使用详情”入口；Kimi Code 0.14.0 已不再支持旧的 `kimi vis --no-open` 本地页面参数。
- 统一需求澄清卡片展开/折叠时的外壳宽度、头部留白和选项列宽，避免展开后选中态和元素宽度看起来跳变。
- 优化 Kimix 兼容 `/custom-theme` 生成提示，强制 18 个 Kimi 主题 token 做语义分层，并在主题导入预览中提示 diff/status/text 层级复用风险。
- 调整 renderer 卡死 watchdog 诊断文件落盘位置，不再自动写到桌面，并限制本地保留数量。

### 侧栏、设置与工作区

- 左侧插件、Hooks、设置入口支持再次点击返回对话页面。
- 统一插件、Hooks、设置等非对话页面的顶栏高度、标题样式、背景和用色方式。
- 设置页分区支持拖拽排序并持久化。
- 右侧会话工具箱卡片支持拖拽排序并持久化，拖动预览统一为线条样式。
- 工具类入口进一步收敛到右侧会话侧栏，减少左下角杂散状态。
- 左侧项目会话列表滚动条更靠近侧栏右边界，减少视觉挤压。

### Git 工具箱

- 优化右侧 Git 工具卡片和详情面板，减少打开详情时的默认文本闪烁。
- Git 状态跟随当前会话对应项目，并在卡片右侧显示项目名，避免误判操作目标。
- 修复有远端项目被显示为无远端、刷新后远端状态不及时的问题。
- 改进无法提交文件的提示方式，避免错误信息大段糊到界面。

### 对话与稳定性

- 修复 Kimi Code 输出正文在部分事件形态下丢失的问题。
- 修复流式输出把 Markdown 表格分隔行或数据行拆碎后，聊天区和导出文件表格渲染错乱的问题。
- 优化聊天区 Markdown 表格顶部留白和全屏宽度，减少宽表格必须横向拖动才能查看的问题。
- 优化折叠的思考、工具和子任务行高度，避免英文下探字符被遮罩或裁切。
- 优化状态更新、上下文压缩、引导和长程任务相关展示，减少内部状态泄露到用户界面。
- 增加卡死诊断日志导出能力，便于定位后台切换或设置页卡顿问题。
- 继续优化消息气泡、代码块、状态行、权限提示和工具调用展示的留白与对齐。
