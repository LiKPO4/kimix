# Kimix v2.11.20

- 修复通过 `find-skills` 等 Skill 安装到 `~/.agents/skills` 后，下一轮对话仍提示当前会话无法识别新 Skill 的问题。
- Kimix 现在会扫描标准 Agent Skill 目录；普通消息发送前会静默同步新安装的顶层 Skill 及其子 Skill 到 Kimi Code。
- 嵌套子 Skill 会在 Kimi Code 同步副本中展平注册，但保留 `game-development/game-design` 这类原始路由名称，避免主 Skill 能加载、子 Skill 仍不可调用。
- 检测到 Skill 比当前注册表更新时，通过官方 fork 保留会话上下文并刷新 Skill 注册表，再发送用户消息。
- 再次打开会话时不再把 `<kimi-skill-loaded>` 和整份内部 Skill 指令显示成用户消息；用户触发只显示简短 `/skill:<name>`，模型自动调用显示为会话状态。
- Skill 作为首轮消息时不再使用 `User activated the skill...` 作为会话标题，已有此类官方标题也会在会话列表中转成简短的“使用 Skill”。
