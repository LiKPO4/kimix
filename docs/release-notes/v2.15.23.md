# Kimix v2.15.23

## 多 Agent 房间单目标阶段

- Composer `+` 菜单新增受内部 gate 控制的“添加 Agent”，直接复用现有 Kimi Code Provider、模型目录和权限配置。
- 普通会话可原地升级为房间；每个新 Agent 拥有独立官方 session、上下文、模型和稳定房间身份，不预设实施者或审查者角色。
- 房间底部新增单接收者选择器，可查看状态、切换目标、重试创建、编辑名称与 `@名称`，或将次要 Agent 移出并保留为独立会话。
- 房间普通消息统一经过可恢复 delivery 事务；`queued`、`sending`、官方接收和 runtime 终态按 Agent 独立结算，快速进入运行态时不会被回退到旧状态。
- 第一阶段一次只允许一个接收者；Agent 工作期间不创建跨 Agent 队列，Slash、Skill、Goal 等 Agent scoped 操作继续保持关闭，避免命中错误 runtime。
- gate 默认关闭；关闭时已存在房间只读可见，普通单 Agent 会话的 UI、发送和恢复链路不变。
