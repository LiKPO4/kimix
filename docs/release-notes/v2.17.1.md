# Kimix v2.17.1

本版本是 **v2.16.88 之后的中型更新**：底座跟进 **Kimi Code 0.29.0**，补齐会话级子 Agent 模型、主 Agent 思考强度、视频多模态、历史用量页脚，并对上一轮全部提交做了一轮系统 review，修复其中发现的 3 项高危与 12 项中危问题。（v2.17.0 未发布即撤版，其全部内容已并入本版本。）正式包仍由 **GitHub Actions** 构建发布。

## 运行时与协议（0.29）

- 本机官方 CLI / vendored Node SDK 底座对齐 **0.29.0**（node-sdk 0.14.0），并保留子 Agent 双模型实验补丁（PR #1996 六提交 + Kimix sticky resume / spawn 审计）。
- 接入官方 **工具 `active` 状态**、**Agent 生命周期诊断**（`agent.created/disposed` + 快照 subagents）。
- 主 Agent **思考强度**可按模型 `support_efforts` / `default_effort` 选择；自定义模型可在设置中**声明**思考档位（官方探测不回档位时以 config 为准），且声明的默认档现在可以真正清除。
- 会话侧栏支持 **子 Agent 默认模型 / 思考强度**；明确独立模型时走兼容路由，并在 fresh runtime 上**重放已存路由**（重启后不再静默回落跟随主模型）；改回跟随主 Agent 时会话**解除兼容路由 pin** 并在下一空闲边界迁回官方 Server 路由（Swarm 会话保留 pin）。
- 强制委派开关：开启后仅在 **wire 发送层**幂等注入委派指令，可见气泡正文不变；官方历史清洗同步剥离该指令，重载/搜索后气泡不显示注入原文。
- Esc / 中断：`prompt.aborted` 纳入等待器，dispatch 立即结算（双路由回归已验证）。

## 安全与凭据

- 已保存的 Provider API Key **仅在同源 Base URL 下复用**：探测模型与测试连接时，若请求 baseUrl 的 origin 与已保存配置不一致，不再把落盘 key 外发，要求显式输入。

## 权限与路由稳定性

- 会话不可用时切换**权限模式**现在真正落盘本地值（此前 toast 提示「下次生效」但实际未保存，收紧权限后下一条消息仍按旧档运行）。
- 子 Agent 路由**重放失败降级为诊断日志并继续发送**，不再阻断发送或被房间投递误标为「结果未知」；下次发送自动重试。
- 移出房间为独立会话时，子 Agent 强制唤起开关随其余路由字段一并镜像，不再丢失。

## 多模态与媒体

- 支持粘贴 / 选择 / 拖入 **视频** 作为官方多模态内容（Server 先上传 file，再以 video+file 发送；兼容 SDK 仍可接收 `video_url`）。
- 视频输入补齐能力回退：模型不支持视频时按图片同款链路剥离为文本占位重试，不再直接报错或误标重试；已有本地数据的视频引用不再按 fileId 重复下载；纯视频消息不再重复显示用户气泡。
- 历史官方视频经 **`kimix-media` + 0.29 `fs:content`（ETag/Range）** 流式播放，失败可回退整段加载；fileId 限制在 `~/.kimi-code/files` 下。

## 对话渲染与用量

- 思考流：合并幂等、快照回放**替换而非追加**；子代理事件不再并入主 turn 草稿，卡片未挂载的子代理事件丢弃并记诊断日志。
- kimi-web 过程展示：思考保持官方内联 ThinkingBlock 风格（与工具组卡区分）。
- 历史轮页脚：从 wire 镜像水合模型 / token；canonical 被拒时增量合并用量状态（含侧栏点开缓存过期会话路径）；头部按**轮级真实模型**显示；历史缓存版本升至 16 强制重新水合。
- Server 快照命中后本地 wire 读取失败不再导致整个会话打不开（降级为未合并的官方快照）。
- 兼容链路会话历史**不再读 Server 休眠快照**，避免「模型请求失败」误报帧。
- 回合 settle 后补挂模型/用量；Composer 上下文环与弹窗与页脚共用合并逻辑；压缩/新轮后的空窗期不再回退显示压缩前的旧窗口。
- 派生事件（diff/change_summary/todo）ID 改用稳定 toolCallId，跨时钟源重放不再产生重复变更统计；历史错位的同源 diff/todo 随 change_summary 一并搬回源轮。

## 性能与稳定性

- 大文件变更统计加 **4000 行阈值回退**：6000 行全量替换的统计耗时从约 2.4s 降到 1ms，不再阻塞流式渲染线程。
- 供应商模型管理面板全部 8 个异步操作补 **try/finally**：IPC 异常不再导致面板按钮永久禁用；**Base URL 探测模型**与添加流程相邻；去掉易混淆的官方目录模型下拉选模。
- 过滤 `models.<alias>.overrides` 伪模型；删除模型时清理残留。
- 健康卡「需关注」可见 Server 徽章与具体项；权限/思考弹窗更紧凑；探测卡与加号/语音按钮圆角对齐。
- 窄宽度工具栏：优先收权限为图标保持一行，再换两行。
- 思考 / 权限 / Plan 在会话不可用时软失败（保留本地值，resume 统一重放）。

## 其他

- 文件变更统计与历史预览补全；重试轮历史隔离；流式正文片段顺序保持。
- 安装列表在 GitHub API 限流时从 `latest.yml` 恢复。
- 知识库补充 runtime 不变量（子 Agent 路由、思考合并、用量水合、0.29 工具/生命周期等）。

## 已知边界（请知晓）

- 子 Agent **独立模型**依赖实验双模型路由；与官方 **自定义 Markdown Agent（agents/*.md）** 在 0.29 上不可兼得（内置 explore/coder/plan 不受影响）。
- 部分轮次官方不提供 `contextTokens` 时，Context 百分比为**基于本轮输入的近似**。
- 子代理路由重放降级后，失败仅诊断日志可见，下次发送时自动重试。
- 被污染的历史会话事件不会自动回刷；请用新会话验证误报/思考重复类问题。
- vendored SDK 仍携带未合并上游 PR #1996；刷新 vendor 需重挑提交。
