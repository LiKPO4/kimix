# Kimix v2.17.1

本版本是 **v2.17.0 的修复版本**：对上一轮全部提交做了一轮系统 review，修复其中发现的 3 项高危与 12 项中危问题，覆盖凭据安全、权限持久化、历史清洗、路由生命周期与渲染性能。正式包仍由 **GitHub Actions** 构建发布。

## 安全与凭据

- 已保存的 Provider API Key **仅在同源 Base URL 下复用**：探测模型与测试连接时，若请求 baseUrl 的 origin 与已保存配置不一致，不再把落盘 key 外发，要求显式输入。

## 权限与路由

- 会话不可用时切换**权限模式**现在真正落盘本地值（此前 toast 提示「下次生效」但实际未保存，收紧权限后下一条消息仍按旧档运行）；发送路径 resume 重放读取到的即为新值。
- 子 Agent 路由**重放失败降级为诊断日志并继续发送**，不再阻断发送或被房间投递误标为「结果未知」（路由是增强不是前提，下次发送自动重试）。
- 清除子 Agent 独立模型/思考配置（改回跟随主 Agent）后，会话**解除 SDK 兼容路由 pin** 并在下一空闲边界迁回官方 Server 路由（Swarm 会话保留 pin）；移出房间为独立会话时强制唤起开关不再丢失。

## 历史与渲染

- 官方历史清洗补齐 **`【强制委派】` 指令剥离**：重载/搜索后用户气泡不再显示注入指令原文，粘贴图片元数据回填与房间消息绑定随之恢复。
- 子代理事件在卡片未挂载时**丢弃并记诊断日志**，不再并入主 turn 正文。
- 压缩/新轮后的空窗期，Composer 上下文环**不再回退显示压缩前的旧窗口**。
- 侧栏点开缓存过期会话（selectSession 慢路径）补齐**用量增量水合与轮级模型回填**，历史缓存版本升至 16 强制重新水合；Server 快照命中后本地 wire 读取失败不再导致整个会话打不开。
- 纯视频消息不再可能重复显示用户气泡；派生事件（diff/change_summary/todo）ID 改用稳定 toolCallId，跨时钟源重放不再产生重复变更统计；历史错位的同源 diff/todo 随 change_summary 一并搬回源轮。

## 模型与视频

- SDK 保存路径现在可以**清除默认思考档**（此前选「不设置」会被旧值静默带回）。
- 视频输入补齐能力回退：模型不支持视频时按图片同款链路剥离为文本占位重试，不再直接报错或误标 non-vision 白重试；已有 dataUrl 的视频引用不再按 fileId 重复下载。

## 性能与稳定性

- 大文件变更统计加 **4000 行阈值回退**：6000 行全量替换的统计耗时从约 2.4s 降到 1ms，不再阻塞流式渲染线程。
- 供应商模型管理面板全部 8 个异步操作补 **try/finally**：IPC 异常 reject 不再导致面板按钮永久禁用。

## 验证

- `pnpm typecheck` 双配置通过；全量 `pnpm test:run` 126 文件 1121 用例通过（含本轮新增约 25 条回归用例）。
- 知识库 `pnpm knowledge:validate` PASS（不变量 73/79 同步修订）。

## 建议重点复验

1. 会话删除/不可用时把权限从 yolo 收紧到默认档，发一条消息确认按新档运行。
2. 开启强制唤起发送后重启应用，用户气泡不应出现 `【强制委派】` 原文。
3. 子 Agent 设独立模型后再改回跟随主 Agent，发一轮确认回到 Server 路由（Markdown 自定义 Agent 恢复可用）。
4. 压缩上下文结束后、下一轮开始前，输入区上下文环不应显示压缩前的旧窗口。
5. 探测模型前把 Base URL 改成非同源地址，确认不再静默复用已保存 key。
