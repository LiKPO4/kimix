# Kimix v2.9.22

## 主要更新

- 刷新内置 Kimi Code node-sdk 到官方 `0.14.0` 对应源码，SDK 包版本为 `0.9.1`。
- 补齐官方 `Interrupt` Hook 事件：类型、主进程校验、规则生成提示和 Hooks 面板下拉项已同步。
- `/swarm` 改为调用官方 SDK：支持 `/swarm on`、`/swarm off` 和 `/swarm <任务>`，不再作为普通消息发送。
- `/swarm <任务>` 会先显式进入官方 Swarm task 模式，并在对话流里显示独立的“已发出 Swarm 指令”提示；Kimix 不再提前插入空助手消息，避免误显示为普通“正在思考”。
- 输入框上方新增 Swarm 子进程悬浮卡，类似 TodoList 展示多个子进程的排队、运行、限流、完成和失败状态；每个子进程会保留官方任务描述，并显示最近的正文、思考、工具调用或状态进展，可收起到右侧会话侧栏。
- 修复 Swarm/引导/导入/设置拖拽等核验问题：TurnEnd 不再误关闭 queued/suspended 子代理，steer wire 确认改为异步读取并超时降级，CC Codex Skill 导入会先备份，历史修复、renderer watchdog 和拖拽监听增加兜底清理。
- 文件附件改为和图片一起显示在附件栏位；发送、排队、引导、重新发送时会把附件文件名和绝对路径写入 prompt，避免 agent 只拿到文件名后再搜索。
- 修复 Swarm 后期无新事件时过程摘要计时停止的问题；运行中的工具或子代理会继续驱动“正在思考/执行中”时长刷新。
- Swarm 子进程悬浮卡只显示最新一批子代理，避免上一轮已完成的子代理项混入当前运行列表。
- 修复启动后左侧一次性出现大量“新会话”加载占位的问题：启动恢复只处理当前项目的可用最新会话，不再把所有最近项目的 SDK 历史会话注入成本地 loading 占位。
- 新增 `docs/kimi-code-0.14-followup.md`，记录 0.14.0 已跟进项、探针结果和后续待接入能力。
