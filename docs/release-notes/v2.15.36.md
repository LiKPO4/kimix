# Kimix v2.15.36

## 多 Agent 房间 Windows 与终态稳定性

- 次要 Agent 的官方 session ID 改为由稳定 Agent 身份派生的 Windows 文件系统安全 ID，避免冒号导致创建失败，同时继续通过 room metadata 保留原始 Agent 归属。
- 多 Agent 并行结束时，状态筛选改为按 `agentTurnId` 分区；一个 Agent 的最终 usage 不再被另一个 Agent 后到达的状态隐藏。
- 连续状态快照合并时保留已有输入、输出和 Context 数值，普通“已完成”快照不会覆盖本轮最终用量。
- SDK 成功终态会短暂等待最终用量事件再广播完成；未提供用量的 Provider 仍通过短 fallback 正常结束。
